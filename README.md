 SQL Practice Session – E-commerce Dataset

# Objective
To strengthen core SQL concepts required for Data Analyst roles by solving structured business-based queries using an e-commerce dataset.

# Dataset Overview
Two tables were used:

1. Customers (c)
customer_id
name
city

2️. Orders (o)
order_id
customer_id
order_date
amount

# Concepts Practiced
SUM()
COUNT()
AVG()
GROUP BY
HAVING
ORDER BY
LIMIT / OFFSET
JOIN
LEFT JOIN
Handling NULL
Ranking logic
Time-based grouping (strftime())

# Questions Solved
1. Show total revenue generated.
2. Find total number of orders per customer.
3. Identify the customer who generated the highest revenue.
4. Find customers who placed more than one order.
5. Calculate total revenue per city.
6. Determine which month generated the highest revenue.
7. Find customers who never placed an order.
8. Identify the second highest spending customer.
9. Create a customer KPI report including:
 -Total orders
 -Total revenue
 -Average order value

# Key Learning Outcomes
-Understood the difference between WHERE and HAVING.
-Learned proper use of LEFT JOIN for identifying missing records.
-Practiced ranking logic for top/second-highest queries.

Built mini KPI-style reporting queries.

Improved thinking in terms of business questions rather than syntax alone.
